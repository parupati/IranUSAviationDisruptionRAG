# Iran-US Aviation Disruption RAG System

A Retrieval-Augmented Generation system built on the [Global Civil Aviation Disruption 2026 Iran-US War](https://www.kaggle.com/datasets/zkskhurram/global-civil-aviation-disruption2026-iranus-war) dataset.

## Prerequisites

- Python 3.12+
- An OpenAI API key ([get one here](https://platform.openai.com/api-keys))

## Project Structure

```
IranUSAviationDisruptionRAG/
  .env                  # Your OpenAI API key (not committed to git)
  requirements.txt      # Python dependencies
  data/                 # 6 CSV files from Kaggle dataset
  chroma_db/            # Local vector store (generated by ingest.py)
  src/
    ingest.py           # Converts CSVs to natural language chunks and builds vector store
    rag.py              # Interactive RAG query engine (retrieval + GPT-4o)
  test_retrieval.py     # Verifies vector store retrieval works (no API key needed)
```

## Setup

### 1. Create and activate the virtual environment

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS/Linux
source .venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
pip install langchain-huggingface langchain-core
```

### 3. Set your OpenAI API key

Edit the `.env` file in the project root and replace the placeholder:

```
OPENAI_API_KEY=sk-your-actual-key-here
```

> **Important:** Never commit your `.env` file. It is already listed in `.gitignore`.

### 4. Download the dataset (if not already present)

If the `data/` folder is empty, download the dataset using the Kaggle CLI:

```bash
pip install kaggle
kaggle datasets download -d zkskhurram/global-civil-aviation-disruption2026-iranus-war -p data --unzip
```

> You need a Kaggle account and API token configured at `~/.kaggle/kaggle.json`. See [Kaggle API docs](https://github.com/Kaggle/kaggle-api#api-credentials).

### 5. Build the vector store

```bash
python src/ingest.py
```

This reads all 6 CSV files, converts each row into a natural language description, generates embeddings using the HuggingFace `all-MiniLM-L6-v2` model (runs locally on CPU), and stores them in a local ChromaDB database under `chroma_db/`.

Expected output:
```
Loading and converting CSV data to documents...
Created 218 document chunks from 6 CSV files
Building vector store (this may take a minute on first run)...
Indexed 218 documents into ChromaDB
Done! Vector store is ready.
```

## Usage

### Run the interactive query engine

```bash
python src/rag.py
```

Type your questions at the prompt. Type `quit` to exit.

### Example queries

- "Which airline suffered the highest daily financial loss?"
- "What airports in Iran were closed?"
- "How many flights were cancelled from Dubai on March 1st?"
- "What was the aviation impact of the Natanz airstrike?"
- "Which flights were rerouted and what was the extra fuel cost?"
- "What countries closed their airspace and for how long?"

### Test retrieval without an API key

To verify the vector store works (no OpenAI key needed):

```bash
python test_retrieval.py
```

This runs two similarity searches against the local vector store and prints the top matching chunks.

## How It Works

1. **Ingestion** (`ingest.py`): Each CSV row is converted into a human-readable sentence. For example, a row from `airline_losses_estimate.csv` becomes: *"Emirates (UAE) faces an estimated daily financial loss of $4,200,000 USD due to the Iran-US conflict..."*

2. **Embedding**: The `all-MiniLM-L6-v2` model (runs on CPU) converts each sentence into a 384-dimensional vector.

3. **Storage**: Vectors are stored in a local ChromaDB database on disk.

4. **Query**: When you ask a question, it is embedded using the same model, and the top-k most similar chunks are retrieved from ChromaDB.

5. **Generation**: The retrieved chunks + your question are sent to OpenAI GPT-4o, which generates a natural language answer grounded in the data.

## Dataset Details

| File | Records | Description |
|---|---|---|
| `airline_losses_estimate.csv` | 35 | Daily financial losses per airline |
| `airport_disruptions.csv` | 35 | Disruptions at major airport hubs |
| `airspace_closures.csv` | 25 | Country-level airspace closures with NOTAM references |
| `conflict_events.csv` | 27 | Timeline of military events and their aviation impact |
| `flight_cancellations.csv` | 47 | Individual cancelled flights with routes and reasons |
| `flight_reroutes.csv` | 45 | Rerouted flights with extra distance, fuel cost, and delay |

## Rebuilding the Vector Store

If you update the data files or change the ingestion logic, delete the `chroma_db/` folder and re-run:

```bash
rmdir /s /q chroma_db   # Windows
rm -rf chroma_db         # macOS/Linux

python src/ingest.py
```
